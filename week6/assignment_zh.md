# 第 6 周 - 使用 Semgrep 扫描并修复漏洞

## 作业概览
使用 **Semgrep** 对 `week6/` 中提供的应用进行静态分析。对分析结果进行分类和判断，并至少修复 3 个安全问题。在你的说明文档中，解释 Semgrep 发现了哪些问题，以及你是如何修复它们的。

## 了解 Semgrep
Semgrep 是一个开源静态分析工具，可以搜索代码、发现缺陷，并强制执行安全防护与编码规范。

1. 点击[这里](https://github.com/semgrep/semgrep/blob/develop/README.md)了解 Semgrep。
2. 按照上述链接中的安装说明进行安装。你可以自行选择使用 **Semgrep Appsec Platform** 或 **CLI 工具**。

## 扫描任务

### 你需要扫描的内容
- 后端 Python（FastAPI）：`week6/backend/`
- 前端 JavaScript：`week6/frontend/`
- 依赖项：`week6/requirements.txt`
- 配置 / 环境文件（检查 secrets）：`week6/` 内的文件

### 运行通用安全扫描，以及针对 secrets 和依赖项的专项扫描
在**作业仓库根目录**下运行以下命令，使用一个经过整理、适用于 CI 的规则集合，其中同时包含代码规则和 secrets 规则：

```bash
semgrep ci --subdir week6
```

## 任务要求
1. 从 Semgrep 识别出的结果中任选 3 个问题，使用你选择的 AI 编码工具修复它们。
2. 展示精确的修改内容，并解释缓解措施（例如参数化 SQL、更安全的 API、更强的加密方式、对 DOM 写入进行清理、限制 CORS、升级依赖版本）。
3. 重要：确保修复后应用仍然可以运行，并且测试仍然通过。

## 交付物
### 1. 简要问题概览
- 总结 Semgrep 报告的问题类别（SAST / Secrets / SCA）。
- 说明你决定忽略的误报或噪声规则，以及原因。

### 2. 三个修复（修改前 -> 修改后）
对于每个已修复的问题，请提供：
- 文件和行号
- Semgrep 标记的规则 / 类别
- 简要风险说明
- 你的修改内容（简短代码 diff 或说明，以及所使用的 AI 编码工具）
- 为什么这个修改能够缓解该问题

## 提示
- 优先做最小且有针对性的修改，从根本原因入手解决问题。
- 每修复一个问题后都重新运行 Semgrep，确认该问题已消失，并且没有引入新的问题。
- 如果你修复的是依赖问题，请记录升级后的版本；如果使用了供应链扫描，也请附上相关安全公告链接。
